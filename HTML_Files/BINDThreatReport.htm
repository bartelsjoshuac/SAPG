1. Spoofing the User Web Application External Entity  [State: Mitigation Implemented]  [Priority: Medium] 

Category:Spoofing
Description:User Web Application may be spoofed by an attacker and this may lead to unauthorized access to OpenLDAP (slapd process). Consider using a standard authentication mechanism to identify the external entity.
Justification:In a typical setup, the web applicaiton would have already performed a BIND as an application ID, and would perform a REBIND with the credentials of the user logging in. It would then discard that connection and create a new one for another user. ACLs would be configured to prevent BINDs but non applicationIDs.

2. Elevation Using Impersonation  [State: Not Applicable]  [Priority: Low] 

Category:Elevation Of Privilege
Description:OpenLDAP (slapd process) may be able to impersonate the context of User Web Application in order to gain additional privilege.
Justification:<no mitigation provided>

3. Potential Data Repudiation by OpenLDAP (slapd process)  [State: Mitigation Implemented]  [Priority: Medium] 

Category:Repudiation
Description:OpenLDAP (slapd process) claims that it did not receive data from a source outside the trust boundary. Consider using logging or auditing to record the source, time, and summary of the received data.
Justification:/var/log/slapd-access should be monitored for abnormal behavior

4. Potential Process Crash or Stop for OpenLDAP (slapd process)  [State: Not Started]  [Priority: High] 

Category:Denial Of Service
Description:OpenLDAP (slapd process) crashes, halts, stops or runs slowly; in all cases violating an availability metric.
Justification:This is a know issue (https://bugs.launchpad.net/ubuntu/+source/openldap/+bug/1838380) and it can be mitigated via systemd

5. Data Flow LDAPS Is Potentially Interrupted  [State: Mitigation Implemented]  [Priority: Medium] 

Category:Denial Of Service
Description:An external agent interrupts data flowing across a trust boundary in either direction.
Justification:LDAP uses TCP/IP vs UDP

6. OpenLDAP (slapd process) May be Subject to Elevation of Privilege Using Remote Code Execution  [State: Needs Investigation]  [Priority: High] 

Category:Elevation Of Privilege
Description:User Web Application may be able to remotely execute code for OpenLDAP (slapd process).
Justification:A malformed request should and would normally be rejected and lookThruLimit and maxResult would help prevent DOS attacks

7. Elevation by Changing the Execution Flow in OpenLDAP (slapd process)  [State: Needs Investigation]  [Priority: High] 

Category:Elevation Of Privilege
Description:An attacker may pass data into OpenLDAP (slapd process) in order to change the flow of program execution within OpenLDAP (slapd process) to the attacker's choosing.
Justification:See Threat #5

Interaction: LDAPS

8. Spoofing of the User Web Application External Destination Entity  [State: Mitigation Implemented]  [Priority: Medium] 

Category:Spoofing
Description:User Web Application may be spoofed by an attacker and this may lead to data being sent to the attacker's target instead of User Web Application. Consider using a standard authentication mechanism to identify the external entity.
Justification:See Threat #1

9. External Entity User Web Application Potentially Denies Receiving Data  [State: Needs Investigation]  [Priority: High] 

Category:Repudiation
Description:User Web Application claims that it did not receive data from a process on the other side of the trust boundary. Consider using logging or auditing to record the source, time, and summary of the received data.
Justification:The application would be configured for failover and fault tollerance to multiple OpenLDAP nodes
